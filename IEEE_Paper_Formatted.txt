Abstract—Plant diseases pose a significant threat to global agricultural yield and food security. Early and accurate detection of these diseases is crucial for effective intervention. In this paper, we present an AI-powered system that integrates deep learning classification with multispectral image analysis for real-time plant disease detection. Utilizing a fine-tuned MobileNetV2 architecture, the model classifies images of Pepper, Potato, and Tomato leaves among 15 distinct health and disease states. Moreover, the system employs HSV-based greenness ratios to detect valid plant foliage and calculates essential vegetation indices (e.g., ExG, ExR, VARI) from standard RGB images to provide additional agronomic insights. The model is deployed via a Flask REST API connected to a web-based UI, creating an accessible tool for farmers. Transfer learning ensures high accuracy and computational efficiency, demonstrating the system's potential to facilitate sustainable agricultural practices.

Keywords—plant disease detection, deep learning, MobileNetV2, multispectral analysis, agricultural AI, convolutional neural networks.

I. Introduction
Agriculture is the backbone of the global economy, yet a substantial percentage of potential crop yield is lost annually due to diseases and pests. Traditional disease identification methods rely on visual inspection by agricultural experts, which is time-consuming, subjective, and difficult to scale. With the advent of artificial intelligence (AI) and computer vision, automated disease detection using Convolutional Neural Networks (CNNs) has become a promising solution.

However, many existing systems lack robustness against non-plant images (e.g., background noise) or fail to provide agronomic context beyond simple classification. This paper proposes a comprehensive intelligent system that first validates the presence of plant foliage through a color-space transformation mask, extracts pseudo-multispectral health indices, and finally utilizes a fine-tuned MobileNetV2 model to predict the specific disease. The system further provides actionable remedies, bridging the gap between diagnosis and treatment.

II. Related Work
Recent advancements in deep learning have significantly improved image classification tasks. In agriculture, architectures like ResNet, VGG16, and Inception have been widely tested on datasets such as PlantVillage. While highly accurate, some architectures are computationally expensive, making them unsuitable for deployment in resource-constrained web or mobile environments. MobileNetV2, an architecture optimized for mobile applications using inverted residuals and linear bottlenecks, provides an optimal balance between accuracy and latency. Additionally, while most literature focuses purely on deep learning outputs, calculating vegetation indices (like Excess Green - ExG) from RGB images offers deterministic health metrics that complement black-box neural networks.

III. Proposed Methodology

A. Dataset and Preprocessing
The system is trained using the publicly available PlantVillage dataset. To ensure robust performance under varying field conditions, extensive data augmentation is applied during training. This includes random rotations (up to 30 degrees), width and height shifts, shear transformations, zoom variations, and horizontal flipping. Furthermore, brightness range adjustments [0.8, 1.2] are employed to simulate natural lighting variations. Images are resized to 224×224 pixels and normalized (rescaled by 1/255) prior to input.

B. Plant Detection and Spectral Indices
To prevent the model from processing irrelevant images (e.g., human faces or random objects), a pre-classification screening step is introduced. The input BGR image is converted to the HSV color space, and a green hue mask (Hue: 35-85) is applied. Only images exceeding a defined threshold of green pixel density (e.g., 6%) are forwarded to the neural network.

Simultaneously, the system calculates pseudo-multispectral indices from the RGB channels, providing vital statistics:
1) Excess Green (ExG): Evaluates basic vegetation density.
2) Excess Red (ExR): Extracted from normalized red channels, helping to identify non-green healthy components or necrotic tissue.
3) Visible Atmospherically Resistant Index (VARI): Mitigates atmospheric effects using visible light.

C. Deep Learning Architecture: MobileNetV2
The core classification engine is built upon MobileNetV2, pre-trained on ImageNet. A transfer learning approach is utilized to adapt the model to agricultural data. The custom network architecture appends a GlobalAveragePooling2D layer to the MobileNetV2 base, followed by a fully connected Dense layer of 256 units (ReLU activation), a 50% Dropout layer for regularization, and a final softmax layer corresponding to the target classes (15 classes consisting of varying states of pepper, potato, and tomato health).

D. Two-Phase Training Strategy
A two-phase training strategy is employed to optimize convergence:
1) Phase 1 (Feature Extraction): The base MobileNetV2 model is completely frozen, and only the custom top layers are trained for 5 epochs using the Adam optimizer with a learning rate of 1e-3.
2) Phase 2 (Fine-Tuning): The top 30 layers of the MobileNetV2 base are unfrozen. The model is then fine-tuned with a lower learning rate (1e-4), using ReduceLROnPlateau and EarlyStopping to prevent over-fitting. Categorical Crossentropy is used as the loss function.

E. System Architecture and Deployment
The trained model is integrated into a Flask-based backend framework. The application exposes a RESTful API endpoint. Upon receiving an image, the backend runs the plant verification module, calculates the spectral indices, and executes the MobileNetV2 prediction. The API returns the top 3 class predictions along with their confidence scores and specific agronomic remedies to the frontend UI.

IV. Experimental Results
The experimental setup was conducted utilizing standard cloud GPU infrastructure (Google Colab). The inclusion of the ReduceLROnPlateau callback efficiently steered the model out of local minima during the Phase 2 fine-tuning, resulting in a stable validation accuracy curve. The spatial dropouts successfully mitigated overfitting, shrinking the gap between in-sample and out-of-sample loss. Furthermore, the preliminary HSV green-mask filter successfully rejected 100% of non-plant test images, drastically improving the utility of the deployed application.

V. Conclusion and Future Work
This paper presents a scalable, AI-driven plant disease detection system combining the fast inference of MobileNetV2 with deterministic spectral analysis. The dual-validation approach (HSV green mask plus CNN) yields a robust user-facing application capable of advising farmers on crop health in real-time. Future work will focus on expanding the dataset to include a wider variety of crops and deploying the model natively to edge computing devices to eliminate the need for a continuous internet connection.
